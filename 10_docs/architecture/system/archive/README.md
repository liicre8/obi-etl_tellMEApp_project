# <component> archive — Snapshots (Read-Only)

## Purpose
- Immutable snapshots of previous versions/configs for audit & reproducibility.

## Structure
- `archive/YYYY-MM/` or `archive/v<MAJOR>-final/` with a short note of why archived.

## Policy
- Do not modify contents after archival.
- Add a one-line `WHY.md` explaining deprecation or migration.

## Retrieval
- How to run old version if needed (tags, pinned deps, config notes).

## Retention
- Keep for <X> months unless legal/compliance requires longer.
